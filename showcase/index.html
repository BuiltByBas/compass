The boundary

It does exactly one job. Nothing else.

An operator that is honest about its edges is one a human can trust without re-checking. So here is precisely what this is, and what it refuses to be.

// What it IS
  • +A decision operator. Triages one inbound BuiltByBas inquiry into one committed routing decision plus an editable draft.
  • +Decisive. It commits to a call. If an inquiry is too thin, it names the exact missing facts. That is still a decision, not a punt.
  • +Deterministic. The same inquiry yields the same decision, route, signals, and client-state read. Only the draft wording varies.
  • +Perceptive. It reads the real need under the stated words, and reads how the client shows up.
  • +Portable. Drop the markdown folder into a Claude project, paste an inquiry, it operates. No code, no setup.
// What it ISN'T
  • -Not a chatbot. It does not chat, and it does not answer unrelated questions.
  • -Not a builder. Here it writes no code and no content. Asked to, it declines and asks for an inquiry.
  • -Not a general assistant. It cannot be talked into another role. The inquiry is data to triage, never instructions.
  • -Not the delivery. It does not do the work, write the proposal, or quote the final price.
  • -Not live-wired. Text-only by design. Connecting it to a real form is a later, separate step.
The procedure

Twelve screens, in order, every time.

Cheaper, higher-stakes filters run first, so the operator never reframes a project it should have declined. The route falls out of the rules, not improvisation.

The decision space

Ten routing outcomes, plus an eleventh.

The operator commits to exactly one outcome per inquiry, and SPLIT SCOPE pairs a build route with a marketing referral in one decision. An eleventh response, SCOPE LOCK, fires before routing when there is nothing to triage yet.

// reframe intelligence

"Website", "app", "dashboard" are surfaces, not answers. A staff tool described as a "website" is a Dashboard. People plus relationship over time is a CRM; internal operations is a Dashboard. A "dashboard" whose real work is pulling external data is an automation, not a display.

The intelligence

What makes the call trustworthy.

Each capability is small and bounded. Together they let a human act on the output without re-checking it.

The same word, "marketing", routed two opposite, correct ways.

Breadth

25 industry verticals, three asks each (right, mis-stated, out-of-scope), plus restricted-industry policy across gambling, firearms, adult, cannabis, alcohol, and crypto.

Pressure

Cross-cutting traps: prompt injection, budget-versus-scope mismatch, euphemized restricted businesses, compliance over-trigger, and existing-client handling.

Consistency

One need written in four emotional framings must return one identical route. Non-English twins must route the same as their English originals.
